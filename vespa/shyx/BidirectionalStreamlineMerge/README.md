# BidirectionalStreamlineMerge (双向流线合并)

作为一名在代码之海中冲浪十年的全栈老兵，我来为您揭秘这个将“劳燕分飞”的流线重新牵红线的神奇模块。

## 1. 目的与功能算法详细解释

**目的**：
在流线追踪（Stream Tracing）中，单个种子点（Seed）通常会向着两个相反方向（前向和后向）发射出两条多段线（Polyline）。这个类的目的就是将这两条本同根生、却各自天涯的流线段“破镜重圆”，合并为一条连续的、完整的单向数据流线，同时清理掉多余的重复节点。

**功能与核心算法**：
1. **种子匹配（Match-making）**：过滤器首先通过指定的 `SeedIds` 数组将所有 `LINE` 或 `POLY_LINE` 单元进行分组匹配。这就像是给每条流线发了DNA检测报告，只有同源的才会放在一起。
2. **反向拼接（Reverse & Stitch）**：对于拥有恰好两条流线段的种子点，算法会默认输入顺序中的**第二条**为后向分支。它会把这条后向分支“倒放”（反转点序），然后无缝拼接到前向分支的前面。
3. **去重切除（Junction Clean-up）**：既然是拼在一起，连接处（初始种子点）就会出现两个重叠的点。算法会精准识别（基于点ID相同或空间坐标严格相等），并把多余的那个点无情踢出局，确保整条流线平滑连续。对于本来就只有单向线段的“单身狗”种子，算法会保持尊重，直接原样复制。
4. **积分与微分计算（Math Magic）**：除了合并，算法还支持沿着顶点顺序计算累积梯形积分（注意：是按顶点索引的积分，不乘以实际几何长度）和后向差分。

## 2. 参数列表及其效果和含义

这里是这个模块的“控制面板”，你可以通过以下参数来调整合并过程的“风味”：

* **`SeedIdsArrayName` (string)**: 
  * *含义*：用于标识同源种子的数组名称。默认值为 `"SeedIds"`。
  * *效果*：告诉算法去哪里寻找流线的DNA标识。
* **`SeedIdsOnCells` (int/bool)**:
  * *含义*：标识种子ID是存储在单元数据(Cell Data)中还是点数据(Point Data)中。默认开启(`1`)。
  * *效果*：开启时读取单元数据（一条线一个ID）；关闭时，算法会去读取每条多段线的**第一个点**的点数据来作为种子ID。
* **`GeneratePointIdArray` (int/bool)**:
  * *含义*：是否生成新的顶点ID数组。默认开启(`1`)。
  * *效果*：如果开启，会在输出中调用 `vtkGenerateIds` 为合并后的流线点生成从 `0` 到 `N-1` 的连续索引。
* **`PointIdsArrayName` (string)**:
  * *含义*：新生成的顶点ID数组的名称。默认值为 `"PointIds"`。
  * *效果*：给重新洗牌后的顶点重新发身份证号。
* **`ComputeArcLengthIntegral` (int/bool)**:
  * *含义*：是否沿着流线计算累积积分。默认关闭(`0`)。
  * *效果*：开启后，会使用梯形法则按**顶点索引**（非几何长度）累加求和。如果是向量则计算其模长。
* **`IntegrandArrayName` (string)**:
  * *含义*：用于积分的输入点数据数组名。
  * *效果*：指定你想把哪份数据用来做“滚雪球”般的积分计算。
* **`IntegralArrayName` (string)**:
  * *含义*：积分结果的输出数组名。默认值为 `"StreamlineIntegral"`。
  * *效果*：积分累加的结果就存放在这个名字的数组里。
* **`ComputeIntegrandDelta` (int/bool)**:
  * *含义*：是否计算沿线的后向差分（相邻点的差值）。默认关闭(`0`)。
  * *效果*：开启后，计算公式为 $f_i - f_{i-1}$。注意第一点会变成 `NaN`（毕竟它前面没人了）。
* **`DeltaArrayName` (string)**:
  * *含义*：差分结果的输出数组名。默认值为 `"StreamlineIntegrandDelta"`。
* **`EnablePerStreamlineRandomOffset` (int/bool)**:
  * *含义*：是否为每条合并后的流线添加一个整体的随机偏移量。默认关闭(`0`)。
  * *效果*：为每条流线整体增加一点“个性”误差（同一条线上的所有点偏移量相同，不同线偏移量不同）。
* **`RandomOffsetArrayName` (string)**:
  * *含义*：需要应用随机偏移的输入点数据数组名称（会被拷贝到输出并加上偏移）。
* **`RandomOffsetRangeMax` (double)**:
  * *含义*：随机偏移的上限 $n$，随机数在 $[0, n]$ 之间均匀分布。默认值为 `1.0`。
  * *效果*：控制这波“随机个性”的强度。设为 $\le 0$ 时偏移即为0。
* **`RandomOffsetSeed` (int)**:
  * *含义*：随机数生成器的种子。默认值为 `0`。
  * *效果*：固定种子可以保证每次生成的随机偏移都是可复现的（拒绝薛定谔的流线）。
