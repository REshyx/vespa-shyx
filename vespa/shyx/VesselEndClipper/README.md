# VesselEndClipper (血管端点裁剪器) ✂️🩸

欢迎来到 `VesselEndClipper` 模块！如果你曾经对着血管提取出来的参差不齐、犹如狗啃一般的末端发愁，并且 CFD（计算流体力学）求解器因为这些不平整的边界条件向你疯狂报错，那么这个模块就是你的救星。

顾名思义，`vtkCGALVesselEndClipper` 是一位优雅的“血管外科医生”，专门负责在血管的末端切出完美、平坦的横截面。

## 🎯 目的与功能算法详细解释

### 核心目的
本模块的主要使命是：接收一个三维血管表面网格及其对应的中心线（骨架），自动找到所有的血管端点（入口和出口），并在这些地方极其利落地“砍”上一刀，生成平滑的切割面。最终，它能为你提供一个适合直接丢进 CFD 软件里进行流体仿真的水密网格（Watertight Mesh）。

### 算法工作流（它是如何下刀的？）
我们的“虚拟手术”分为以下几个精密步骤：

1. **骨架探路 (Graph Construction):** 首先，算法会遍历你输入的中心线数据，构建一个点与点之间的邻接图（Adjacency Graph）。
2. **定位端点与瞄准 (Endpoint Detection & Tangent Estimation):** 算法在图中寻找度数为 1（degree-1）的“孤独”节点，这些就是血管的末端。接着，从每个末端顺藤摸瓜向内走几步（步数由 `TangentDepth` 决定），计算出一条局部切线。这条切线就像是狙击枪的红外线，决定了我们将要下刀的**裁剪平面法向量**。
3. **空间切割与智能保底 (Topology-aware Clipping):** 这是算法最聪明的地方。我们用计算出的无限大平面把整个网格切成两半。但是等等！万一这个无限大的平面不小心把远处弯过来的另一根血管也给切了怎么办？别慌！算法会进行**连通性分析 (Connectivity Analysis)**，只精准移除距离端点最近的那个小小的“尖端 (Tip)”，而把同一侧其他无辜被切断的血管片段重新缝合回去。
4. **伤口缝合 (Capping):** 既然切开了，总不能留个洞。如果开启了 `CapEndpoints` 功能，算法会贴心地用多边形把切口补平，确保整个血管依然是一个天衣无缝的封闭模型。

## 🎛️ 参数列表及其效果和含义

在 ParaView 面板中，你可以通过调整以下旋钮来指挥这位“外科医生”：

| 参数名称 | 默认值 | 含义与效果 |
| :--- | :--- | :--- |
| **`ClipOffset` (裁剪偏移量)** | `0.0` | 决定下刀的具体位置。正值意味着把刀往血管外部挪（切得少一点），负值意味着把刀往血管内部捅（切得深一点，常用于避开末端畸形区域）。 |
| **`TangentDepth` (切线探测深度)** | `1` | 范围 `[1, 10]`。算法向内探查的骨架边数。数值越大，计算出的切向就越综合、越平滑；数值越小，切向就越贴近最末端局部的“拐角”情况。如果你的血管末端扭曲得像麻花，建议把这个值调大一点。 |
| **`CapEndpoints` (端点封口)** | `true` | 是否给切出的伤口贴上“创可贴”。勾选此项后，被切开的横截面会被自动用平面填充。进行 CFD 仿真前**强烈建议**保持开启。 |
| **`EndpointSelection` (端点选择列表)** | - | 这是一个动态生成的勾选框列表，列出了所有被发现的端点。你可以通过复选框精准地告诉算法：“切掉端点 1 和 2，但放过端点 3”。 |
