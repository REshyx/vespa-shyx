# RadiusNeighborCount (固定半径邻域点计数)

## 🎯 目的与功能
`vtkSHYXRadiusNeighborCount` 是一个基于 VTK (Visualization Toolkit) 的点云数据处理算法（继承自 `vtkPolyDataAlgorithm`）。它的主要目的非常纯粹，用一句话概括就是：**“告诉我，在这个圈子里，到底有多少个兄弟？”**

具体而言，给定一个 `vtkPolyData` 输入，该算法会遍历其中的每一个点，并在以该点为圆心、指定 `Radius` 为半径的闭球域内（包含查询点自身），精确统计出存在多少个邻居点。

处理完成后，算法不会破坏你原有珍贵的数据结构，而是巧妙地进行一次“浅拷贝”（Shallow Copy），并贴心地在输出结果的点数据（Point Data）中挂载一个新的数组——`NeighborsInRadius`（`vtkIdTypeArray` 类型）。每个点对应的邻居数量就安安静静地躺在这个数组里，供你随时查阅，非常适合用于局部密度估计、异常点剔除前的特征提取等场景。

## 🧠 算法原理解析
如果你以为它只是用两层 `for` 循环暴力计算两点之间的距离，那你就太小看它了（毕竟那样的话，CPU 可能会冒烟 💨）。为了让性能飞起来，本算法祭出了两大“法宝”：

1. **高效空间索引 (`vtkStaticPointLocator`)**：
   在正式开始“查户口”之前，算法会先给所有的点建立一个高效率的静态点定位器。你可以把它理解为给点云建立了一本详细的“空间黄页”。有了它，我们就不需要遍历全图找邻居，而是直接通过空间分块快速锁定潜在的目标群体。
2. **多线程并发加速 (`vtkSMPTools`)**：
   面对海量的点云数据，单挑是不明智的，必须群殴！算法利用了 VTK 的对称多处理工具 (`vtkSMPTools::For`)。它把所有点的查询任务切分成多个批次，交给多个 CPU 核心同时开工。不仅如此，还通过线程本地存储 (`vtkSMPThreadLocalObject<vtkIdList>`) 给每个线程分配了独立的“记事本”，完美避免了多线程抢夺资源的尴尬。

## ⚙️ 参数列表及其含义

| 参数名 | 类型 | 默认值 | 效果与含义 |
| :--- | :--- | :--- | :--- |
| **`Radius`** | `double` | `1.0` | **邻域搜索半径**。这是本算法唯一也是最核心的参数。它定义了每个点“社交圈”的物理大小。<br> - **效果**：该值越大，框住的邻居自然越多（当然计算量也会相应增加）；该值越小，每个点就越显得“孤僻”。<br> - **约束**：必须是一个大于 `0.0` 且有限的（Finite）正数，否则算法会傲娇地报错并拒绝工作。 |

*(注：输出数据中的新数组固定命名为 `NeighborsInRadius`，该名称硬编码在算法中，作为结果提取的唯一凭证。)*
