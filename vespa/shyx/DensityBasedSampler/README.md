# DensityBasedSampler (基于密度的点云采样器)

欢迎来到 `DensityBasedSampler`！这里是让你能够随心所欲在封闭网格“体内”撒点的地方。

## 🎯 目的与功能算法详细解释

### 目的与功能
本模块的核心组件是 `vtkCGALDensityBasedSampler`，它的主要目的是**在封闭网格内部生成随机点云，且点云的分布密度受控于你提供的空间标量场**。
就像你在草坪上撒种子，土壤肥沃（标量值高）的地方多撒点，贫瘠（标量值低）的地方少撒点。它既能处理体网格（如 `.vtu` 格式的 `vtkUnstructuredGrid`），也能处理封闭的表面网格（`vtkPolyData`）。输出结果是纯粹的顶点几何体（点云）。

### 算法流程
它的魔法分为以下几个步骤：
1. **外壳提取**：无论你喂给它体网格还是表面，它都会先检查，如果是体网格，它会使用 `vtkDataSetSurfaceFilter` 剥掉内部结构，只保留最外层的“皮”（表面网格）。
2. **构建笛卡尔晶格**：根据网格的包围盒（Bounding Box）比例和参数 `PreSampleCount`，它会建立一个致密的三维网格坐标系，生成一大批均匀分布的“候选点”。
3. **内外大筛查（预采样）**：利用 `vtkImplicitPolyDataDistance` 计算每个“候选点”到表面的有向距离。只有距离 $\le 0$（也就是老老实实在模型肚子里的点）才会被视为“内部点”留下来，其他在模型外面的点直接被无情淘汰。
4. **密度俄罗斯轮盘赌（核心采样）**：
   - 算法会读取你指定的 `DensityArrayName`，并通过 `vtkCellLocator` 进行空间插值，获取每个“内部点”所在位置的标量值。
   - 将标量值的整个数据范围线性映射为 $0\% \sim 100\%$ 的概率（Survival Probability）。
   - 最后，请出随机数发生器生成 $[0, 1]$ 之间的随机数。如果随机数小于该点的存活概率，点被留下；否则就地“牺牲”。（这就是经典的接受-拒绝采样法）
   - *（当然，如果你没有提供密度数组，它会退化为“均匀采样”，跳过这一步，直接把所有内部点打包带走。）*
5. **多线程火力全开**：为了防止大规模点云生成卡爆，底层在评估内部点和插值时大量使用了 `vtkSMPTools` 进行多线程并发加速（前提是编译时开启了 `VESPA_USE_SMP`）。

## ⚙️ 参数列表及其效果和含义

要想用得爽，参数得调好。以下是全部可配置参数大赏：

| 参数名 | 类型 | 默认值 | 效果与含义 |
| :--- | :---: | :---: | :--- |
| **`PreSampleCount`** | `int` | `100000` | **预采样网格点数。** 这是撒点的“弹药基数”。它决定了生成三维笛卡尔网格的总点数。注意，这**不是**最终输出的点数！因为有很多点会在模型外部被剔除，或者在密度判定中被刷掉。值越大，候选点越密，结果越精细，但 CPU 也会燃烧得更旺。*(取值范围: 1 - 100000000)* |
| **`DensityArrayName`** | `std::string` | `""` | **密度数组名称。** 用于控制空间密度的点数据（Point-Data）标量数组名称。该数组的值越大，生成点的概率越高。如果你把它留空，或者传入 `"(Uniform)"`，算法会自动退化为**均匀采样**，也就是众生平等，每个网格内部点都被保留。 |
| **`Seed`** | `int` | `0` | **随机数种子。** 控制那场“俄罗斯轮盘赌”的伪随机数发生器。如果你希望每次运行代码得到的点云长得一模一样（为了极佳的可复现性和可控的测试），请固定这个种子参数。 |
