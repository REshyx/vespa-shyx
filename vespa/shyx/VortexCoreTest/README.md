# VortexCoreTest 核心算法解析指南

各位科学计算的修仙者们，欢迎来到 `VortexCoreTest` 的源码结界！如果你在流场的混沌之中寻找传说中的“涡核（Vortex Core）”，那么这个类就是你的寻龙尺。

## 1. 目的与功能算法详细解释

**目的**：
`vtkVortexCoreTest` 是一个用于在三维向量场（通常是流体速度场）中提取涡核中心线的过滤器（Algorithm Filter）。它是基于 VTK 的 `vtkVortexCore` 和平行向量算法（Parallel Vectors Algorithm）深度定制而成的“VESPA 专属版”。简单来说，它的任务就是在大风大浪的流场中，精准定位并提取出龙卷风的中心骨架！

**功能与算法核心**：
算法的核心思想是寻找空间中两个特定物理量“相互平行”的区域，从而将涡核以几何折线（Polylines）的形式具象化。它打出的是一套流体力学与数值计算的“两步走”组合拳：

1. **场量计算与平行向量提取**：
   - **基础模式**：计算速度场（Velocity, $v$）和加速度场（Acceleration, $a = J \cdot v$，其中 $J$ 为速度的雅可比矩阵）。算法会去寻找这两个向量场方向平行的位置，这是经典涡核提取的精髓。
   - **高阶模式（Higher Order Method）**：如果觉得基础款不够精准，开启高阶模式后，算法会计算“加加速度”（Jerk，即加速度的导数）。这涉及到雅可比矩阵的梯度（一个包含27个分量的三阶庞大张量）。此时，算法寻找的是速度场 $v$ 和 Jerk 场平行的位置。

2. **涡旋准则过滤（Vortex Criteria Filtering）**：
   仅仅向量平行还不够，还要满足严苛的涡旋流体力学标准，防止把普通的流线误认为涡核。算法会使用多线程对网格点进行“四重考验”：
   - **Q-Criterion ($Q > 0$)**：要求流场的旋转率必须战胜应变率。
   - **$\Delta$-Criterion ($\Delta > 0$)**：确保速度梯度张量的特征方程拥有复数解。
   - **$\lambda_2$-Criterion ($\lambda_2 < 0$)**：识别局部压强极小值区域。
   - **$\lambda_{ci}$-Criterion**：提取出真实的旋转强度（即特征值的虚部）。
   
   通不过这些考验的点，统统会被无情抛弃（`acceptedPoints` 标记为 false）。最后，算法还会顺手帮你计算并输出涡量大小（Vorticity Magnitude）。

## 2. 参数列表及其效果和含义

想要优雅地玩转这个类，你需要了解它的两大法宝（参数）：

| 参数名称 | 类型 | 默认值 | 效果和含义 |
| :--- | :---: | :---: | :--- |
| `HigherOrderMethod` | `bool` | `false` | **高阶方法开关**。关闭时，算法寻找速度场与**加速度场**平行的线；开启时，算法将深入高阶求导，计算 27 分量的雅可比梯度，寻找速度场与**Jerk（加加速度）场**平行的线。<br>**效果**：开启后能捕获更复杂、更精细的涡核结构，但会让你的 CPU 疯狂燃烧，适合追求极致精度的“强迫症”玩家。 |
| `FasterApproximation` | `bool` | `false` | **快速近似开关**。作用于底层的 `vtkGradientFilter`（梯度计算器）。<br>**效果**：开启后，梯度的计算会采用更粗略但更快速的近似差分算法。如果你想要“马上看到结果”以验证管道，可以将其打开；如果追求严谨的科学计算和高颜值的结果，请务必保持关闭。 |
