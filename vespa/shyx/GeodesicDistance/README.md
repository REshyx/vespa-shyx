# Geodesic Distance Filter (测地线距离计算器)

欢迎来到 **Geodesic Distance Filter** 的文档！在这个模块中，我们将带你从一个点出发，以最短的路径“跋山涉水”游遍整个表面网格。无论你的模型表面有多么崎岖不平，这个滤镜都能帮你找到两点之间最接地气的路径。

## 🎯 目的与功能算法详细解释

### 目的
在三维网格处理中，两点之间的欧几里得距离（也就是常说的直线距离）往往具有欺骗性。就像在现实中，你不能直接穿透一座大山到达山背面的村庄，你得顺着山坡爬过去！`vtkGeodesicDistanceFilter` 的主要任务就是计算**表面测地线距离**（Geodesic Distance）——即沿着网格表面，从一个指定的**源顶点（Source Vertex）**出发，到达模型上所有其他顶点的最短路径长度。

### 功能与算法原理
我们怎么知道哪条路最近呢？这里请出了图论界的大明星：**Dijkstra 算法**（迪杰斯特拉算法）。
1. **数据准备**：首先，滤镜会检查你输入的是不是正经的 `vtkPolyData`（必须包含多边形面片数据，光有几个孤零零的点可不行）。
2. **图的构建与搜索**：底层依赖于 `vtkDijkstraGraphGeodesicPath`。它会将你的三维表面网格转化为一个庞大的图（Graph），网格的边就是图的边，边的权重则是边长。
3. **全局波前传播**：滤镜关闭了“找到终点就停止”的设定（`StopWhenEndReachedOff`），这意味着 Dijkstra 算法会像水波一样从源顶点向外蔓延，直到计算出到达图中所有连通顶点的最短距离。
4. **输出结果**：最终，滤镜会在输出数据的点属性（Point Data）中，贴心地加上一个名为 `GeodesicDistance` 的数据数组，并将其设为默认激活的标量。对于那些像孤岛一样永远无法到达的顶点，距离会被无情地标记为 `-1.0`。

## 🎛️ 参数列表及其效果和含义

要使这个滤镜顺利运转，你只需要提供一个核心参数：

| 参数名称 | 数据类型 | 含义与效果 | 默认值 |
| :--- | :--- | :--- | :--- |
| **`SourceVertexId`** | `vtkIdType` (整数) | **源顶点 ID**。<br>这就是我们的“出发点”。你必须指定一个有效的顶点索引。如果你的模型有 N 个顶点，这个值必须在 `[0, N-1]` 之间。如果设置越界，滤镜会直接报错罢工并返回 0。 | `0` |

*注：除此之外，模块为标准 VTK 滤镜管线（1个输入端口，1个输出端口），底层会自动接管所有复杂的权重累加与节点分配。*
