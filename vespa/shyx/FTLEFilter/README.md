# 🚀 FTLEFilter: 有限时间李雅普诺夫指数计算器

嗨，朋友！欢迎来到 `FTLEFilter` 的世界。如果你在茫茫的流场数据中迷失了方向，不知道粒子们要去哪儿、从哪儿来，那你算是找对工具了。本滤镜旨在通过计算**有限时间李雅普诺夫指数 (Finite-Time Lyapunov Exponent, 简称 FTLE)**，帮你精准定位流场中的“交通拥堵区”和“高速分流区”——也就是高大上的**拉格朗日拟序结构 (Lagrangian Coherent Structures, LCS)**。

## 🎯 目的与功能算法详细解释

### 目的
简单来说，`FTLEFilter` 就是流场中的“显微镜”和“水晶球”。通过追踪粒子的运动轨迹，它能揭示流体内部隐藏的骨架（排斥结构和吸引结构）。正向时间积分可以找到排斥结构（粒子四散逃离的地方），逆向时间积分能找到吸引结构（粒子聚拢的归宿）。

### 算法流程
别看名字长得吓人，这兄弟的工作原理其实非常脚踏实地，具体步骤如下：
1. **铺设赛道 (Velocity Interpolation)**：首先获取输入数据中的速度场矢量，并利用 `vtkCompositeInterpolatedVelocityField` 构建一个可以在任何位置插值速度的平滑赛道。
2. **放手去飞 (Particle Advection)**：在空间中的每个网格点上放一个虚拟粒子。选择你偏爱的数值积分器（RK2 或 RK4），按照设定的时间步长（`StepSize`），让粒子顺流而下（或者逆流而上），飞行一段指定的时间（`IntegrationTime`）。
3. **记录终点 (Flow Map Generation)**：粒子停下后，记录下它们现在的坐标，这就构成了所谓的“流图” (Flow Map)。
4. **探求变化 (Jacobian & Cauchy-Green Tensor)**：使用 `vtkGradientFilter` 计算流图的空间梯度（雅可比矩阵 $F$），接着用它算出柯西-格林变形张量 $C = F^T F$。这玩意儿听着玄乎，其实就是用来衡量“原本挨在一块的两个粒子，现在被扯得多开”。
5. **计算真身 (FTLE Calculation)**：提取张量 $C$ 的最大特征值 $\lambda_{max}$。最终的 FTLE 值便呼之欲出：
   $$ FTLE = \frac{1}{|T|} \ln(\sqrt{\lambda_{max}}) $$
   算完后，它会把这个值贴心挂在每个点的数据上，命名为 `FTLE`，大功告成！

## 🎛️ 参数列表及其效果和含义

这台精密的仪器给你留了几个旋钮，以下是“操作指南”：

* **`IntegrationTime` (积分时间)**: 
  * *含义*：粒子随波逐流的总时长。
  * *效果*：决定了你能看到多大尺度的结构。正值表示正向 FTLE（寻找排斥结构），负值表示逆向 FTLE（寻找吸引结构）。注意，时间太短看不出结构，时间太长粒子可能会飞出边界。
* **`StepSize` (步长)**:
  * *含义*：数值积分时的时间步长。
  * *效果*：控制计算精度和时间的平衡。步长越小，结果越精准，但你的 CPU 也会抗议得更大声；步长太大可能会导致粒子运动轨迹失真。
* **`StartTime` (起始时间)**:
  * *含义*：粒子释放的绝对时间。
  * *效果*：主要在 `Pathline`（迹线）模式下使用。流场如果随时间变化（非定常流），你需要告诉粒子该从哪一秒开始它的旅程。
* **`IntegratorType` (积分器类型)**:
  * *含义*：计算粒子运动轨迹的数学方法，可选 `INTEGRATOR_RK2` (二阶龙格库塔) 或 `INTEGRATOR_RK4` (四阶龙格库塔)。
  * *效果*：RK4 精度更高，是推荐的默认老大哥；RK2 速度更快，适合对精度要求不高或需要快速预览的场景。
* **`AdvectionMode` (平流模式)**:
  * *含义*：假设流场是否随时间变化。可选 `MODE_STREAMLINE` (流线，假设定常流) 或 `MODE_PATHLINE` (迹线，假设非定常流)。
  * *效果*：如果你的流场是冻结在某一时刻的，用 Streamline；如果是随时间不断变化的真实流场，必须用 Pathline，否则粒子会被带偏。
* **`VelocityArrayName` (速度数组名)**:
  * *含义*：输入数据中代表速度的数组名称。
  * *效果*：如果不填，系统会努力自动寻找活跃的矢量场；如果填错了，它就会罢工并给你甩出一个 Error。
