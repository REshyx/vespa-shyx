# 🦴 SkeletonExtraction (3D骨架提取)

欢迎来到 `SkeletonExtraction` 模块！如果你曾经看着一个丰满的3D模型，并在心里默默想：“这家伙到底有没有骨气（骨架）？”，那么这个模块就是为你准备的。

## 🎯 目的与功能算法详细解释

**目的**： 
本模块的核心功能是从一个闭合的（Watertight）3D三角表面网格中，无情且精准地“榨取”出它的一维曲线骨架（1D Curve Skeleton）。输出结果为包含多段线（Polylines）的 `vtkPolyData`。

**算法与流程**： 
这可不是靠黑魔法，而是依赖于强大的 **CGAL Mean Curvature Flow Skeletonization（平均曲率流骨架化）** 算法。整个工作流如下：
1. **类型转换大挪移**：首先，将输入的 VTK 格式数据转换为 CGAL 能看懂的表面网格（Surface Mesh）。
2. **“漏水”检测**：严格检查模型是不是闭合的（不闭合的模型是不配拥有骨架的！）。
3. **疯狂收缩（Contraction）**：在平均曲率流的作用下，网格开始不断向内收缩，就像是用魔法吸干了模型的水分，直到它干瘪成一条条骨干。
4. **图结构转换与输出**：将 CGAL 提取出的骨架图结构（顶点和边）重新打包成 VTK 多段线形式（`vtkPoints` 和 `vtkLine`），并优雅地呈现给你。

## 🎛️ 参数列表及其效果和含义

要想榨出完美的骨架，火候（参数）的控制至关重要。以下是你可以调节的旋钮：

| 参数名 | 类型 | 默认值 | 含义与效果 |
| :--- | :---: | :---: | :--- |
| **MaxTriangleAngle** | `double` | 110.0 | **最大三角形角度（度）**。在局部重网格化（Remeshing）时，角度大于此值的“过于尖锐”的三角形将被无情拆分，保证网格质量。 |
| **MinEdgeLength** | `double` | 0.0 | **最小边长**。局部重网格化时，短于该值的边会被折叠。设为 `<= 0` 时会聪明的采用 CGAL 默认的网格自适应值。 |
| **MaxIterations** | `int` | 500 | **最大迭代次数**。收缩步骤的最高循环次数。如果收缩太慢，它可以及时喊停，防止程序“思考人生”太久。 |
| **AreaThreshold** | `double` | 1e-4 | **面积阈值**。网格收缩时可达到的最大表面积比率。当收缩后的面积小于这个阈值时，算法就会心满意足地停止。 |
| **QualitySpeedTradeoff** | `double` | 0.1 | **质量与速度的权衡 (w_H)**。值越小，收敛速度越快，但提取出的骨架可能稍微“粗糙”一点；想追求艺术品般的骨架？调大它。 |
| **MediallyCentered** | `bool` | true | **是否居中**。开启后会计算严格居中于内部的骨架。虽然会慢一点，但为了好看，等待是值得的！ |
| **MediallyCenteredSpeedTradeoff**| `double` | 0.2 | **居中平滑度权衡 (w_M)**。仅在开启居中时生效。值越高，生成的骨架越逼近精确的内侧轴，不过同样会拖慢一点收缩的速度。 |
