# vtkSHYXDisconnectedRegionFuse

欢迎来到 `vtkSHYXDisconnectedRegionFuse` 的魔法世界！这里专门负责给破碎的表面网格（Surface Mesh）做“缝合手术”。

## 1. 目的与功能算法详细解释

**目的与功能**：
在三维网格处理中，我们经常会遇到本该是一体的模型，却因为各种原因裂成了几个互不相连的独立区域（Disconnected Regions）。本类的核心功能就是**将这些断开的区域重新缝合在一起**。它的高明之处在于：**只融合属于不同区域且距离相近的顶点，绝不动同一个区域内部的顶点**。这样既能把碎块拼凑完整，又不会导致模型内部原本的精细结构糊成一团。一个区域的顶点可以与多个其他区域的顶点进行多对多的融合。

**算法详细步骤**：
1. **区域标记 (Region Labeling)**：首先召唤 `vtkPolyDataConnectivityFilter`，为输入的网格提取所有的连通区域，并给每个顶点打上专属的 `RegionId` 标签。
2. **分组归类 (Grouping)**：根据 `RegionId`，将顶点按区域分门别类，装进不同的“桶”里备用。
3. **跨区匹配 (Cross-Region Matching)**：使用高性能的 `vtkStaticPointLocator` 在不同区域之间进行空间近邻搜索。如果来自区域 A 和区域 B 的两个顶点之间的距离小于设定的阈值，它们就会被标记为需要合并的候选对（支持 VTK SMP 多线程并行加速，找得飞快！）。
4. **并查集聚类 (Union-Find Clustering)**：为了处理错综复杂的合并关系（比如 A 和 B 连，B 又和 C 连），算法使用并查集（Union-Find）数据结构将它们归入同一个等价类。聚类后，新的合并顶点的坐标取该族群内所有旧顶点的**重心（平均位置）**。
5. **拓扑重构 (Topology Update)**：遍历原有的多边形面片（Polygons），将面片中的旧顶点 ID 替换为新的统一 ID。在这个过程中，如果发现面片因为顶点融合而发生了退化（比如顶点数少于 3 个，缩成了一根线或一个点），就会被无情地剔除。
6. **属性继承 (Attribute Mapping)**：最后，更新 `PointData` 和 `CellData`。新合并的顶点会随机应变地继承其等价类中第一个旧顶点的属性，保留原始面片的单元数据。

## 2. 参数列表及其效果和含义

本类遵循“少即是多”的设计原则，只暴露了一个但极其核心的参数：

* **`FuseThreshold`** (类型: `double`, 默认值: `0.01`)
  * **含义**：融合距离阈值。
  * **效果**：它定义了“多近才算一家人”。只有当来自**不同区域**的两个顶点之间的欧氏距离小于或等于 `FuseThreshold` 时，它们才会被拉到一起合并。
  * **注意事项**：设得太小，可能缝合不上（亲人相见不相识，裂缝依然存在）；设得太大，可能会把不该连的区域强行粘在一起（乱拉红线），导致模型严重扭曲穿模。请务必根据你模型的实际物理尺度来谨慎调节它！
