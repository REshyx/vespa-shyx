# TetGen (vtkSHYXTetGen)

## 🎯 目的与功能算法详解 (Purpose & Algorithm)

各位老铁，欢迎来到三维网格生成的魔法世界！`vtkSHYXTetGen` 是一个专门用于将**封闭表面网格（表面只有一层皮）**变成**四面体体积网格（里面充满了肉）**的VTK过滤器。

**核心目的**：
它的主要任务是接收一个封闭的、且必须是水密（watertight）的三角面片数据（`vtkPolyData`），并在其内部生成实心的四面体网格数据（`vtkUnstructuredGrid`）。如果你给它一个破洞的网格，它可能会当场罢工，所以请务必确保你的输入模型“滴水不漏”。

**底层算法**：
这个小家伙背后站着的是大名鼎鼎的 **TetGen** 库。它主要使用了**约束Delaunay四面体化 (Constrained Delaunay Tetrahedralization, CDT)** 算法。
简单来说，它的工作流程是这样的：
1. **解析输入**：把VTK格式的表面三角面片数据，翻译成TetGen能听懂的底层数据结构。
2. **内部填充**：利用Delaunay算法在三维空间中寻找最优的点连接方式，确保生成的四面体尽可能接近正四面体（极力避免出现过于扁平或尖锐的“劣质”网格）。
3. **约束保持**：在保证四面体质量的同时，还得尽量保住你原本输入的面片形状不变（取决于你是否允许它在表面增加新的顶点）。
4. **输出结果**：将TetGen吐出来的结果，重新组装回VTK标准的非结构化网格格式。

## ⚙️ 参数列表及其效果与含义 (Parameters)

想要完美驯服这个四面体生成器，你需要掌握以下几个神奇的旋钮：

| 参数名 | 默认值 | 对应的TetGen开关 | 含义与效果 |
| :--- | :---: | :---: | :--- |
| **MaxVolume** | `0.0` | `-a` | **最大四面体体积限制**。如果设置大于0，算法会强制细分，确保没有任何一个四面体的体积超过这个值。设为0则任由它“野蛮生长”。 |
| **MaxRadiusEdgeRatio** | `1.8` | `-q` | **最大半径边缘比**。用于控制网格质量（外接球半径与最短边长的比值）。值越小，四面体越接近正四面体。开启时必须 `≥1.2`。设为0表示不作限制。 |
| **MinDihedralAngle** | `0.0` | `-q` | **最小二面角（度）**。同样用于控制网格质量，避免产生过于扁平的面。开启时必须在 `0` 到 `90` 度之间。设为0表示不作限制。 |
| **Nobisect** | `ON` | `-Y` | **保持表面原样**。如果开启，算法绝不会在你的输入表面上添加新的顶点（Steiner points），保证你的原表面“原汁原味”。 |
| **UseCDT** | `OFF` | `-D` | **使用约束Delaunay细分**。注意，这个选项与 `Nobisect` 是死对头（互斥）。开启后可以在保证约束的前提下对网格进行质量优化。 |
| **CDTRefine** | `7` | `-D#` | **CDT细分级别**。范围从1到7，仅在 `UseCDT` 开启时有效，数字越大细分越猛。 |
| **DoCheck** | `OFF` | `-C` | **网格一致性检查**。开启后，TetGen会在生成完毕后自我体检，看看有没有交叉或重叠的糟糕网格。 |
| **Epsilon** | `1e-8` | `-T` | **共面测试容差**。浮点数运算难免有误差，这个值用来判定两个面是不是“差不多在同一个平面上”。 |
