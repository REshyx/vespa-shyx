# SurfaceTipExtractor 尖端提取器

## 🎯 目的与功能算法详细解释

**目的**： 
`vtkSurfaceTipExtractor` 就像是 3D 模型世界的“寻龙尺”，专门用来寻找那些“鹤立鸡群”的尖端或锐利特征（比如树冠的顶点、山峰的尖塔，或者是扎手的多边形刺）。它通过为模型表面（`vtkPolyData`）的每个顶点计算一个“锐度得分（TipScore）”，来量化每个点的尖锐程度。得分越高，说明这个点越“扎手”；得分越低，说明这个点越平滑或者平坦。

**算法流程**：
它的核心算法逻辑简单而优雅，我们可以称之为“找重心、量距离”大法：
1. **画圈找邻居**：对于表面上的每一个顶点 $P_i$，以它为球心，以 `SearchRadius`（搜索半径）画一个球，利用 Point Locator 找出所有落在这个球内的候选顶点。
2. **关系户筛选（测地线连通性）**：仅仅在球内还不够，必须得是“顺藤摸瓜”能连上的。算法通过广度优先搜索（BFS），沿着网格的边（Edges）和面（Cells）寻找那些真正与 $P_i$ 在拓扑上连通的邻居点，形成一个**局部测地线连通邻域**。
3. **计算引力中心**：把上一步找出来的所有连通邻居点聚在一起，计算它们的几何中心（Centroid / 重心）。
4. **得出结论**：计算原始顶点 $P_i$ 到这个重心之间的欧氏距离。如果这个点是在一个平坦的平面上，重心大概率就和点本身重合了（距离近乎为0）；但如果这个点是一个尖锐的凸起，由于邻居们都向下方延伸，重心就会被拉低，导致顶点到重心的距离变大！这个偏离距离就是最终输出的 `TipScore`。

## ⚙️ 参数列表及其效果和含义

目前这个过滤器提供了一个极其关键的旋钮，它决定了提取器的“眼界”有多宽：

| 参数名 | 类型 | 默认值 | 含义与效果 |
| :--- | :--- | :--- | :--- |
| `SearchRadius` | `double` | `5.0` | **搜索半径**。定义了计算每个顶点局部邻域时所使用的球体半径。<br><br>👉 **效果说明**：<br>- **设得过大**：就像在宏观尺度上观察，能找到巨大的山顶，但可能会把一些微小的尖刺忽略掉。同时，计算范围变大，BFS 的耗时也会相应增加。<br>- **设得过小**：变成“显微镜模式”，对网格的微小起伏和噪点极其敏感。如果半径比网格的平均边长还小，它可能找不到任何邻居，得分全变成 0。<br>💡 *调参建议：请根据你的 3D 模型的实际空间尺度（Bounding Box）和你想要提取的特征大小，来灵活调整这个值。* |

*(注：输出结果是一份与输入结构完全相同的 PolyData，但会在 PointData 中新增一个名为 `TipScore` 的双精度浮点数组，请记得查收！)*
