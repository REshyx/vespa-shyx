# Array Probability Point Cull (基于数组概率的点云剔除)

## 目的与功能算法详细解释

**目的**：
`vtkSHYXArrayProbabilityPointCull` 的主要目的是基于输入数据集的点标量数据（Point Data Scalars），对点云进行概率性降采样（Subsampling）。这就像是给每个点发了一张“生存彩票”，中奖概率就写在点的数据里，没中奖的点只能遗憾退场。

**功能与算法流程**：
1. **读取概率**：算法会从指定的点数据数组（Weight Array）中读取每个点的标量值 $v$（仅使用第一个分量）。
2. **截断处理（Clamping）**：算法将标量值严格限制在 $[0, 1]$ 区间内，即计算出的保留概率 $p = \max(0, \min(1, v))$。**注意这里没有任何全局归一化操作**！这意味着如果你的标量值是 0.3，那就是实打实的 30% 保留概率。如果值是 100，截断后就是 1（100% 保留）；如果是负数，截断后就是 0（直接淘汰）。
3. **随机抽样**：
   - 对于每个点，生成一个 $[0, 1]$ 的随机数。如果随机数 $< p$，该点就被保留。
   - **多线程魔法 (SMP 支持)**：当启用多线程（`VESPA_USE_SMP`）时，为了保证多线程下结果的**绝对可重复性**，算法没有使用普通的随机数生成器，而是极其聪明地使用了一个基于 `(Seed, pointId)` 的 `SplitMix64` 确定性随机流。这意味着不管线程怎么调度，只要种子和点 ID 没变，每次降采样的结果都分毫不差。
   - **单线程后备**：如果没有启用 SMP，则老老实实地使用 VTK 自带的 `vtkMinimalStandardRandomSequence` 进行线性随机抽样。
4. **输出生成**：最终幸存下来的点会被打包成一个仅包含顶点单元（Vertex Cells）的 `vtkPolyData` 对象，并且继承原点云的所有属性。

如果数组名称为空或者为 `"(Uniform)"`，算法会仁慈地给所有点发放免死金牌（100% 保留）。

---

## 参数列表及其效果和含义

| 参数名 | 类型 | 默认值 | 效果与含义 |
| :--- | :--- | :--- | :--- |
| **WeightArrayName** | `std::string` | `""` | **权重数组名称**。<br>指定用于读取保留概率的点数据标量数组名。如果留空或设置为 `"(Uniform)"`，算法将跳过概率判定，无条件保留所有点。找不到指定数组时也会抛出警告并退化为全保留模式。 |
| **Seed** | `int` | `0` | **随机种子**。<br>控制随机过程的种子，确保在科研或调试时结果可复现。在开启多线程 (SMP) 的情况下，该种子与每个点的全局 ID 结合，生成独一无二的哈希值来决定命运，真正做到“我命由我不由天（由算法决定）”。 |
