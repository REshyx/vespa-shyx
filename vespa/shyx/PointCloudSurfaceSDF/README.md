# PointCloudSurfaceSDF (点云到曲面有符号距离场计算)

欢迎来到 `PointCloudSurfaceSDF` 的核心！这里是一个轻量级但极具威力的 VTK 滤波器，它专门用于解答一个直击灵魂的几何问题：“这些点到底离曲面有多远？它们是在外面蹭蹭，还是已经进去了？”

## 1. 目的与功能算法详细解释

### 🎯 目的与功能
本模块的核心目的，是计算一个离散点云到某个参考三维曲面的**有符号距离场（Signed Distance Field, SDF）**。
给定两份数据——一份是无家可归的“点云”（Point Cloud），另一份是作为地标的“参考曲面”（Reference Surface），本算法会贴心地为每一个点计算出它与曲面的最短空间距离，并带上正负号（正数代表在曲面外，负数代表在曲面内）。最终结果会以一个名为 `SDF` 的数组，直接塞回点云的点属性（PointData）里，方便后续的渲染可视化、碰撞检测或网格重建。

### 🧠 算法解析
在这个看似简单的外表下，跳动着 VTK 稳定且暴力的几何计算心脏：
1. **核心引擎**：算法内部召唤了 `vtkImplicitPolyDataDistance` 大法师。它通过建立**单元定位器（Cell Locator）**来在茫茫空间中快速找到距离每个点最近的网格面片。
2. **伪法线加权（Angle-weighted pseudonormals）**：这不仅是个高级词汇，更是算法聪明的体现。利用这种技术，参考曲面甚至**不需要是完美的闭合流形**。如果网格自带面法线（Cell Normals），它会直接拿来用；如果没有，VTK 会当场现算。它能巧妙处理模型边缘和尖锐折角处的距离符号判定，减少伪影。
3. **多线程火力全开**：为了防止计算海量点云时你的 CPU 在“摸鱼”，算法中使用了 `vtkSMPTools::For`。它会将点云数据切分成多个小块，并在多个线程上同时发起求值（EvaluateFunction）攻击，极大地缩短了高密度点云距离场的计算时间。

## 2. 参数列表及其效果和含义

作为一个成熟且高冷的底层 C++ 模块，它没有暴露出繁琐的 Set/Get 参数，而是遵循“数据即参数”的极简 VTK 管道过滤哲学：

| 端口/方法名称 | 数据类型 / 作用 | 效果与含义 |
| :--- | :--- | :--- |
| **Input Port 0** <br>*(点云数据)* | `vtkPolyData` | 待测的散乱点云集。你可以把它想象成一群排队测距的观众，等待着被打上 SDF 的距离标签。必须提供。 |
| **Input Port 1** <br>*(SetSourceConnection)* | `vtkPolyData` | 参考曲面。这必须是一个带有面片（Cells）的网格数据。如果传个只有点没有面的“空壳”进来，程序会无情地在控制台抛出 `Surface has no cells.` 的报错并拒绝工作。 |
| **输出数组 (SDF)** | `vtkFloatArray` | 计算的最终产物。在输出结果的 `PointData` 中会新增（或覆盖）一个名为 `SDF` 的单通道浮点数组。每个点对应一个 float 距离值。 |
